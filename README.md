1. 우선순위
- 1. 전체적인 CRUD 구현 -> 기본적으로 유저, 성적, 학기에 대한 정보에 대한 CRUD 구현
- 2. 성적 관련 매커니즘 추가하여 SubjectGradeInformation 구현
- 3. 회원가입 추가
- 4. 학사 정보 관련하여 성적에 관한 사항 처리하기
